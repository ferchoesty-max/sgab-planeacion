# Atributos de Calidad de Software

## 1. Seguridad (Confidencialidad)
**Justificación:** El sistema maneja datos sensibles (calificaciones y datos personales). Es crítico que un alumno no pueda modificar sus propias notas.
**Consecuencia de fallo:** Pérdida de credibilidad de la institución y problemas legales por filtración de datos.

## 2. Usabilidad
**Justificación:** Los docentes y alumnos no son necesariamente expertos técnicos. La interfaz debe ser intuitiva para reducir el tiempo de capacitación.
**Consecuencia de fallo:** Rechazo al uso del sistema y aumento en solicitudes de soporte técnico.

## 3. Disponibilidad
**Justificación:** El sistema debe estar accesible 24/7, especialmente en periodos de cierre de semestre.
**Consecuencia de fallo:** Retrasos en la entrega de actas finales y quejas administrativas.
